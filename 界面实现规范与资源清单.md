# 暖桥 Demo · 界面实现规范与资源清单

> **用途**：在你确认后，按《[家长端底部导航-产品方案.md](家长端底部导航-产品方案.md)》改界面时，**提前准备素材**与**对照代码是否规范**。  
> **结论先说**：底部三 Tab **可以不新增任何背景图**，仅用 **Material Icons + 现有主题** 即可实现；下图均为 **加分项 / 品牌化**。

---

## 1. 设计基准（与工程一致）

| 项 | 约定 |
| --- | --- |
| 逻辑宽度参考 | 约 **360 dp**（见 `res/values/dimens.xml` 注释：1260×2800 px、560 dpi 预览器） |
| 页面水平内边距 | **`wb_padding_screen`（20dp）**，新屏面对齐该值 |
| 可点区域高度 | 不低于 **`wb_touch_min`（56dp）**（底部导航项、主按钮） |
| 圆角 / 间距 | 与现有 `Card`、`WarmPrimaryButton` 视觉一致；新增组件优先 **复用** 已有 shape |
| 色彩与字号 | **Material 3** + `ui/theme` 中 `Color` / `Type` / `Theme`，**禁止**在业务 Composable 里硬编码 `#RRGGBB`（除占位色等与 `AssetPhoto` 一致的小范围例外） |

---

## 2. 素材是否必须？

| 类型 | 是否必须 | 说明 |
| --- | --- | --- |
| **底部导航图标** | 否 | 使用 **`androidx.compose.material.icons`**（Filled：`Home`、`Whatshot`、`Person` 等） |
| **顶部装饰插画** | 否 | 无图时用 **渐变 / 纯色** 或与现首页一致的无图布局 |
| **空状态 / 错误插画** | 否 | 可用 **文字 + 图标** 代替 |
| **现有 assets 插画** | 建议保留 | 见 §4；缺失时 `AssetPhoto` 会显示占位色，不崩溃 |

**若你希望视觉更「成品」**，可按 §4 生成 PNG 放入 `android/app/src/main/assets/photos/`。

## 3. 实现时将用到的组件（Compose）

以下为计划使用或扩展的 **Material3 / 导航** 组件，便于你对照设计稿：

| 用途 | 组件 / API | 备注 |
| --- | --- | --- |
| 底部三 Tab | `NavigationBar` + `NavigationBarItem` | 3 项；与 `Scaffold` 的 `bottomBar` 组合 |
| Tab 内多页面 | `NavHost` + `composable` | 家长 `parent/*`、孩子 `child/*` 前缀（与方案书 §4.2 一致） |
| 全局二级页 | 现有 `feed/*`、`detail/*`、`reminder`、`share` | 详情 **隐藏底栏**（`Scaffold` 不含 `bottomBar` 或独立全屏） |
| 家长极简首页 | `Column` + `FilterChip` / `FilterChip` 行 + `WarmPrimaryButton` / `TextButton` | 去掉三张 `Card` 入口 |
| 今日热点顶栏 | `PrimaryTabRow` / `Tab` + `TabRow` 或 **Segmented 风格** 自定义 | 与家长三频道 / 孩子两频道一致 |
| 列表 / 详情 | 现有 `FeedScreen`、`DetailScreen` | 尽量少改内部，由导航传入参数 |
| 我的孩子页 | `ListItem` 或简单 `Row` + `Divider` | 与 Material 列表规范一致 |

**不引入** 新第三方 UI 库（除非后续单独评审）。

## 4. 资源清单（可选 PNG，放 `assets/photos/`）

命名 **小写 + 下划线**，格式 **PNG（支持透明）或 JPG**。宽图建议 **横向 1080～1440 px** 即可（会按 `ContentScale.Crop` 裁切）。

| 文件名（建议） | 用途 | 建议画面内容 | 优先级 |
| --- | --- | --- | --- |
| `role_select_hero.png` | 选角页插画 | 桥、家庭、暖色（已有说明见 `assets/photos/README.txt`） | 已有规范 |
| `reminder_dialog_header.png` | 提醒弹窗顶图 | 暖光、轻抽象 | 已有规范 |
| `parent_header_decoration.png` | 家长 **极简首页** 问候区下方窄横幅装饰 | 轻插画/渐变纹理，**不要**抢 Chip 与按钮视觉 | 可选 |
| `child_home_header.png` | 孩子 **首页** 顶部装饰 | 「分享给父母」氛围，轻快 | 可选 |
| `hot_topics_header.png` | 「今日热点」Tab 顶栏下装饰（若需要） | 报纸/热点抽象，低对比 | 可选 |
| `mine_header.png` | 「我的」顶部个人区背景 | 简洁、大留白 | 可选 |
| `empty_feed_illustration.png` | 列表无数据 | 温和插画 + 后续用文字说明 | 可选 |
| `error_network_illustration.png` | 网络错误空状态 | 与 `humanizeNetworkError` 文案搭配 | 可选 |

**在代码中引用**：在 `WbAssetPhotos`（或同类 object）里 **增加常量路径**，与 `AssetPhoto()` 一致；**勿**把大图放进 `drawable` 以免 APK 膨胀（除非需多密度切图且体积小）。

## 5. 代码规范（改界面时请对照）

### 5.1 结构与命名

- **包路径**：新屏放在 `ui/screens/` 或 `ui/parent/`、`ui/child/`（若拆分）；**避免** 根包散乱文件。
- **可组合函数**：`PascalCase`，与屏幕名一致，如 `ParentHomeTabScreen`、`HotTopicsTabScreen`。
- **路由字符串**：集中定义（如 `object WbRoutes` 或单文件 `NavigationRoutes.kt`），**禁止** 在多处手写 `"parent/home"` 易拼错。

### 5.2 字符串与无障碍

- **用户可见中文** → `res/values/strings.xml`，Composable 内 `stringResource(R.string.xxx)`。
- **图标、插图** → 必须 `contentDescription`；装饰性图可 `contentDescription = null` 并在父级合并语义（按 Material 指南）。

### 5.3 状态与导航

- **兴趣多选**：方案要求与热点 Tab 同步 → 使用 **`ViewModel` + `StateFlow`** 或 **`rememberSaveable` + 提升至 Nav 层**；持久化用现有 `RolePreferences` / DataStore 扩展，**禁止** 仅 `remember` 导致进程杀死后丢失（二期可做）。
- **返回栈**：从 `detail` 返回应回到 **进入前的列表**；`popBackStack` 行为改完后手动测三轮（家长 / 孩子 / 切换 Tab）。

### 5.4 孩子端频道

- **不展示「孩子推荐」**：热点 Tab 内 **仅渲染两个 Segment**；路由上不 `navigate` 到 `feed/child/...`（除非内部工具测试）。

### 5.5 主题与深色模式

- 当前 Demo 以 **浅色暖色** 为主；若新增表面，使用 **`MaterialTheme.colorScheme`** 对应槽位，避免夜间模式下不可读。

### 5.6 网络与安全

- **API Key** 仍在服务端；客户端 **不得** 新增硬编码密钥。
- 明文 HTTP 仅 Demo；注释中可提醒「上架需 HTTPS」。

## 6. 验收检查清单（你可用来对照 PR / 本地改完自测）

- [ ] 家长：首页 **无** 三条大卡，仅有问候 + Chip +「去看今日热点」+ 温情提醒（及方案允许的小文字链）。
- [ ] 家长：热点 Tab **三频道**；孩子：热点 Tab **两频道**（无孩子推荐）。
- [ ] 孩子：首页 **分享 + 提醒** 清晰；底栏三 Tab 与家长 **同构**。
- [ ] 详情页：**底栏隐藏**；系统返回键行为正确。
- [ ] 新增文案均在 `strings.xml`；主要按钮 **56dp** 级触控友好。
- [ ] 无 `.env` / Key 进仓库；新增图片仅在 `assets/photos/` 且体积合理。
