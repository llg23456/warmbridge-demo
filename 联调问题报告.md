# 暖桥 Demo · 客户端与后端联调问题报告

## 1. 现象摘要

| 现象 | 说明 |
| --- | --- |
| 浏览器访问 `http://<电脑IP>:8000/health` 返回 `{"status":"ok"}` | 说明本机 `uvicorn` 已监听 `0.0.0.0:8000`，局域网内可达。 |
| App 内「热点列表」等请求失败，红字报错 | 多为 **请求未真正发出** 或 **URL/策略** 与浏览器不一致。 |
| 报错含 `CLEARTEXT communication to x.x.x.x not permitted by network security policy` | **Android 网络安全策略拦截 HTTP 明文**，与「后端是否正常」无关。 |

---

## 2. 根因分析（按优先级）

### 2.1 HTTP 明文被拦截（已修复方向）

- **原因**：`AndroidManifest` 虽设 `usesCleartextTraffic="true"`，但若同时配置了 `networkSecurityConfig`，则以 **XML 为准**。原配置仅在 `domain-config` 中放行 `10.0.2.2`、`localhost` 的明文；**真机访问 `10.70.x.x` 等局域网 IP 不在白名单内**，系统直接拒绝连接。
- **表现**：OkHttp 抛出 `IOException`，信息中含 `CLEARTEXT ... not permitted`。
- **处理**：将 `res/xml/network_security_config.xml` 改为 **`base-config cleartextTrafficPermitted="true"`**（Demo 用途）。上架生产应改为 **HTTPS** 并收紧域名。

### 2.2 API_BASE_URL 与真机环境不一致

- **原因**：默认 `BuildConfig.API_BASE_URL` 为 **`http://10.0.2.2:8000/`**（官方模拟器访问开发机）。**实体机不会把 `10.0.2.2` 指向你的电脑**。
- **处理**：在 **`android/local.properties`** 中配置（与 `app` 目录同级）：

  ```properties
  warmbridge.api.baseUrl=http://<你电脑的局域网IP>:8000/
  ```

然后 **Sync Gradle、Rebuild、重装 App**。  
 注意：电脑若多网卡 / DHCP，IP 可能变化；浏览器能打开的地址应与 `API_BASE_URL` **一致**（例如日志里曾出现 `10.70.50.248` 与 `10.70.90.210` 不一致时，需统一为当前跑后端的机器 IP）。

### 2.3防火墙或未监听 `0.0.0.0`

- **原因**：仅 `127.0.0.1` 监听时，外机无法访问。
- **处理**：启动命令使用 `uvicorn app.main:app --host 0.0.0.0 --port 8000`；Windows 防火墙放行 **Python / 8000 端口**。

### 2.4 其它说明

- 浏览器请求 **`/favicon.ico` 返回 404**：FastAPI 未提供该路径，**可忽略**。
- 服务端 **`GET /health`** 与 **`GET /api/tags`、`GET /api/feed`** 为不同路径；前者通只说明端口通，App 仍依赖 **`/api/*`** 与正确 `baseUrl`。

### 2.5 `/api/explain` 返回200 但界面仍是「离线 / 未配置密钥」

- **原因一（设计行为）**：`POST /api/explain` 在无密钥或调用蓝心失败时，仍返回 **HTTP 200** 与 JSON 占位内容（`plain_summary` 里含「离线演示模式」；`background` 区分 **未读到 `VIVO_APP_KEY`** 与 **已配置但调用失败**）。**不能**用「200 = 已走通大模型」判断，应看响应体里的 `background` 或字段 `from_llm`（`true` 表示本次为蓝心成功解析）。
- **原因二（解读缓存）**：服务端对「无追问」的解读会 **按 `item_id` 缓存**；**曾修复为仅缓存 `from_llm=true` 的成功结果**。若你仍在旧版本上：第一次在无密钥时点开过某条新闻，会把离线占位写入缓存，之后即使配好密钥、仍不传 `question`，也会一直拿到旧占位。**处理**：升级后端后重启 `uvicorn`；或对该条点一次「带着追问重新解读」（带 `question` 时不走缓存）；或重启后端清空内存缓存。
- **AppKey 以 `=` / `==` 结尾**：赛事下发的 AppKey 常为 Base64 风格，**末尾填充符是正常格式**，不要把 `==` 删掉，也不要在 `.env` 里加引号截断；整行 `VIVO_APP_KEY=sk-xuanji-...==` 即可。

## 3. 建议排查顺序（复现问题时）

1. **先定「唯一正确 IP」**：用手机浏览器打开 `http://x.x.x.x:8000/health` 能通的那个 `x.x.x.x`，就是 App 里必须配置的地址。若曾用过 `10.70.50.248` 而浏览器实际通的是 `10.70.90.210`，说明电脑换网 /多网卡 / DHCP 变了，**App 仍连旧 IP 会报 `Failed to connect`**，与后端是否启动无关。
2. 手机浏览器再测：`http://<该IP>:8000/api/tags` 是否 JSON 正常。
3. 确认 `android/local.properties` 里 `warmbridge.api.baseUrl=http://<该IP>:8000/`（含末尾 `/`），然后 **Sync Gradle → Rebuild**，并**卸载旧 App 后重装**。
4. Logcat 过滤 `OkHttp` / `Retrofit`，查看实际请求 URL 与异常类型。

## 4. 本次工程侧改动记录（摘要）

- **网络安全**：`network_security_config` 放行全局 HTTP 明文（Demo）。
- **多标签**：家长端兴趣标签支持多选；请求以 **`|`** 连接多个标签，服务端 `filter_by_tag` 按标签集合过滤并集。
- **错误文案**：`humanizeNetworkError` 对 CLEARTEXT 类错误增加简要说明。
- **`/api/explain`**：仅缓存蓝心成功结果（`from_llm=true`），避免「先离线后配 key」长期命中错误缓存；响应体增加 `from_llm` 便于区分占位与真模型输出。
