# 通俗视频生成 · 实现备忘

> MVP-C（2026-06）：不下载整段原视频；**og 封面 + 文字解读压缩口播稿 + TTS + ffmpeg Ken Burns** 合成约 30～60 秒 MP4。  
> 与《功能优化方案》中「首版不做整段音视频」并存：本能力为 **P2 通俗视频** 独立迭代。

---

## 1. 用户流程

1. 孩子在「分享」提交 B 站 / 抖音链接 → 家长「孩子推荐」打开详情。
2. 家长先看 **AI 讲给长辈听**（`POST /api/explain`，缓存不变）。
3. 不满意再点 **通俗视频生成**（仅孩子推荐 / 快解析 + 抖音·B 站链接）。
4. 进入 **通俗视频解读** 页：蒙版 + 橙色进度条 + 步骤文案；可返回，任务在服务端继续。
5. 完成后：播放 MP4、分享视频（系统选择 QQ/微信）、下载到相册、可选分享原链接（纯文本）。
6. 页面下方复用 **ExplainPanel**（追问 Chip、听摘要、带着追问重新解读）。
7. **我的 → 通俗视频任务** 列表进入历史任务（含进行中 / 失败）。

## 2. 后端流水线（5 步）

| step | progress | 说明 |
| --- | --- | --- |
| `prepare` | 5% | 读条目；刷新 og 封面；下载封面（带 Referer）；失败则无字品牌占位图（**不用 drawtext 中文**） |
| `script` | 30% | 取/生成 explain；LLM 将 `plain_summary` 压成 30～60 秒口播稿（不改解读缓存） |
| `tts` | 55% | `vivo_tts.synthesize_wav` |
| `merge` | 80% | ffmpeg：封面 Ken Burns + AAC → `server/data/popular_videos/{job_id}.mp4` |
| `done` | 100% | `video_url` = `/api/video/popular/files/{job_id}.mp4` |

**Logcat / 服务端日志**：统一 tag **`WbVideoGen`**（含 `job_id`、`step`、`progress`、失败 `error_step`）。

### 接口

| 方法 | 路径 |
| --- | --- |
| GET | `/api/video/popular/eligible/{item_id}` |
| POST | `/api/video/popular/start` body `{ "item_id" }` |
| GET | `/api/video/popular/{job_id}/status` |
| GET | `/api/video/popular/jobs` |
| GET | `/api/video/popular/files/{job_id}.mp4` |

任务存 **内存**（`popular_video_store.py`），**重启 uvicorn 清空**；成片文件在 `server/data/popular_videos/`。

### 环境依赖

- **ffmpeg** 在 PATH（合成与占位图）
- **VIVO_APP_KEY**（口播稿 LLM + TTS；无密钥时口播降级为截断摘要，TTS 可能失败）
- 手机访问 `video_url` 须用 **局域网 IP**（见 README §2）

## 4. 与图一「全链路」的差异（刻意未做）

- 不下载原视频、不做 ASR、不做图生视频。
- 不服务端存 MP4 给「App 内发家人」；仅 **share_page_url** 传原链接，成片依赖 **本机下载后** 系统分享或 **短期 file URL**。

## 5. 已知画面问题（方框字）

- 现象：橙色底 + **白色空心方框** → ffmpeg `drawtext` 无中文字体。  
- 处理：占位图已改为 **无文字** 品牌底图；封面下载增加 **Referer**。  
- 总览见 [暖桥-开发现状与速查.md](暖桥-开发现状与速查.md)。
